# Deployment & Transfer Guide\n\n## Overview\n\nThis guide covers deploying the Sage Healthcare Platform wireframe navigator and transferring the repository to the mitchdesigns-com organization.\n\n## Quick Deployment (Vercel)\n\n### Option 1: Deploy from GitHub\n\n1. **Connect to Vercel**\n   - Go to [vercel.com](https://vercel.com)\n   - Click \"New Project\"\n   - Import from GitHub: `ojja/sage-wireframe-navigator`\n   - Vercel will auto-detect Next.js configuration\n\n2. **Configure Build Settings**\n   ```bash\n   # Build Command (auto-detected)\n   npm run build\n   \n   # Output Directory (auto-detected)\n   .next\n   \n   # Install Command (auto-detected)\n   npm install\n   ```\n\n3. **Deploy**\n   - Click \"Deploy\"\n   - Your wireframe will be live in ~2 minutes\n   - Get shareable URL for stakeholder reviews\n\n### Option 2: Deploy via CLI\n\n```bash\n# Install Vercel CLI\nnpm i -g vercel\n\n# Deploy from project directory\ncd sage-wireframe-navigator\nvercel\n\n# Follow prompts\n# Select: Yes to deploy\n# Select: mitchdesigns-com (if available)\n# Select: sage-wireframe-navigator\n# Select: No to modify settings\n```\n\n## Alternative Deployment Options\n\n### Netlify\n\n1. **Connect Repository**\n   - Go to [netlify.com](https://netlify.com)\n   - \"New site from Git\" → GitHub\n   - Select repository\n\n2. **Build Settings**\n   ```bash\n   Build command: npm run build\n   Publish directory: out\n   ```\n\n3. **Configure for Static Export**\n   - The project is already configured with `output: 'export'`\n   - Static files will be generated in `/out` directory\n\n### GitHub Pages\n\n1. **Configure Repository**\n   ```bash\n   # Add to next.config.ts\n   const nextConfig = {\n     output: 'export',\n     basePath: '/sage-wireframe-navigator',\n     assetPrefix: '/sage-wireframe-navigator/',\n   }\n   ```\n\n2. **Deploy Action**\n   ```yaml\n   # .github/workflows/deploy.yml\n   name: Deploy to GitHub Pages\n   on:\n     push:\n       branches: [ main ]\n   jobs:\n     build-and-deploy:\n       runs-on: ubuntu-latest\n       steps:\n         - uses: actions/checkout@v3\n         - uses: actions/setup-node@v3\n           with:\n             node-version: '18'\n         - run: npm install\n         - run: npm run build\n         - uses: peaceiris/actions-gh-pages@v3\n           with:\n             github_token: ${{ secrets.GITHUB_TOKEN }}\n             publish_dir: ./out\n   ```\n\n## Repository Transfer to mitchdesigns-com\n\n### Step 1: Transfer Repository\n\n**Method A: GitHub Web Interface**\n1. Go to repository Settings\n2. Scroll to \"Danger Zone\"\n3. Click \"Transfer ownership\"\n4. Enter: `mitchdesigns-com/sage-wireframe-navigator`\n5. Confirm transfer\n\n**Method B: Contact Organization Admin**\n1. Ask mitchdesigns-com admin to invite you as collaborator\n2. Fork repository to organization\n3. Update all references to new URL\n\n### Step 2: Update Repository URLs\n\nAfter transfer, update:\n\n```bash\n# Update remote origin\ngit remote set-url origin https://github.com/mitchdesigns-com/sage-wireframe-navigator.git\n\n# Update package.json\n{\n  \"repository\": {\n    \"type\": \"git\",\n    \"url\": \"https://github.com/mitchdesigns-com/sage-wireframe-navigator\"\n  }\n}\n```\n\n### Step 3: Update Documentation\n\n- Update README.md clone URLs\n- Update CONTRIBUTING.md repository references\n- Update any hardcoded GitHub URLs in documentation\n\n## Environment Configuration\n\n### Environment Variables\n\n```bash\n# .env.local (for local development)\nNEXT_PUBLIC_SITE_URL=http://localhost:3000\nNEXT_PUBLIC_ANALYTICS_ID=your-analytics-id\nNEXT_PUBLIC_ORGANIZATION=mitchdesigns-com\n\n# .env.production (for deployment)\nNEXT_PUBLIC_SITE_URL=https://sage-wireframes.vercel.app\nNEXT_PUBLIC_ANALYTICS_ID=your-analytics-id\nNEXT_PUBLIC_ORGANIZATION=mitchdesigns-com\n```\n\n### Vercel Environment Variables\n\n1. **In Vercel Dashboard**\n   - Go to Project Settings\n   - Environment Variables tab\n   - Add variables for Production, Preview, Development\n\n2. **Required Variables**\n   ```\n   NEXT_PUBLIC_SITE_URL = https://your-domain.vercel.app\n   NEXT_PUBLIC_ANALYTICS_ID = (optional)\n   ```\n\n## Custom Domain Setup\n\n### Vercel Custom Domain\n\n1. **Add Domain**\n   - Go to Project Settings → Domains\n   - Add: `wireframes.sagehealthcare.sa` or `sage-wireframes.mitchdesigns.com`\n\n2. **DNS Configuration**\n   ```dns\n   # CNAME record\n   wireframes CNAME cname.vercel-dns.com\n   \n   # Or A record\n   @ A 76.76.19.61\n   ```\n\n3. **SSL Certificate**\n   - Auto-provisioned by Vercel\n   - Includes automatic renewal\n\n## Testing Deployment\n\n### Pre-deployment Checklist\n\n- [ ] All pages load without errors\n- [ ] Navigation works correctly\n- [ ] Forms render properly (even without backend)\n- [ ] Images and placeholders display\n- [ ] Mobile responsiveness verified\n- [ ] Console shows no critical errors\n- [ ] Lighthouse performance score >90\n\n### Post-deployment Testing\n\n```bash\n# Test critical user flows\n1.
